Translation Quality Assessment

Apple VisionExcellent

Near-perfect OCR input — zero meaning loss in translation. Only noise: UI icon numerals occasionally read as text (e.g. "0 1" in inventory screens), harmless for a translator.

PaddleOCRInconsistent

Perfect on simple single-column Zelda dialogue. Catastrophic on multi-column UI layouts (tent tutorial 0.846 CER, gym challenge 0.783 CER) — word order scrambles completely, translator produces nonsense.

Windows OCRGood

Strong on single-column game dialogue. Minor character confusions (しや→じゃ, そ・→ぞ) rarely break translation. Multi-column UI text struggles with word ordering. Best speed/accuracy ratio on Windows at 257ms.

RapidOCR-AGood

Single character substitutions (含→今, 清→着, チニ→チュ) still produce real Japanese words — a translator handles these gracefully. One hard failure on yellow-outlined paraglider text. Overall meaning is preserved in >85% of images.

MangaOCR-colPoor

Slightly better than binarised variant (70% vs 66%) but still hallucinates on dark game backgrounds. MangaOCR was trained on white-background manga panels — Zelda dark scenes confuse it fundamentally.

MangaOCRDangerous

The worst failure mode: hallucinates plausible-sounding but semantically wrong Japanese (「あれは母が通りつけ出した」instead of「あれは塔が地面からせり出した」). A translator produces fluent English with the wrong meaning — harder to detect than obvious garbage.

RapidOCR-BUnusable

CLAHE adaptive binarisation produces completely garbage output on dark game backgrounds. Untranslatable on most game images. Retire this preprocessing approach.

RapidOCR-CUnusable

Morphological background subtraction destroys thin kana strokes. Produces ~50% garbage across the board. No translation possible from this output.
